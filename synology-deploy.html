Synology Deployment Guide

Balance Acupuncture — hosting the website on a Synology NAS

Overview

What you'll need

This website is entirely static HTML — no database, no backend framework. It runs on any web server that can serve files. On a Synology NAS, the simplest approach is the Web Station package with either Apache or Nginx.

Step 1

File structure

Your complete website folder should look like this. All files sit at the root — no subdirectories needed for the core pages.

balance-acupuncture/ <-- this is your web root folder ├── index.html <-- homepage ├── about.html ├── services.html ├── herbal.html ├── conditions.html ├── fees.html ├── contact.html ├── privacy.html ├── 404.html ├── style.css <-- shared stylesheet │ └── wix-embeds/ <-- optional, can leave out of web root ├── wix-guide.html ├── wix-head-code.html └── embed-*.html
Tip — keep wix-embeds out of the public web root. The wix-embeds/ folder is a working reference, not part of the live site. Either exclude it when uploading, or leave it — it won't cause any harm if it's there.
Step 2

Install Web Station

  1. Log in to your Synology DSM interface (typically http://your-nas-ip:5000)
  2. Open Package Center from the main menu
  3. Search for Web Station and click Install
  4. When prompted, also install Nginx HTTP Server 1.x (or Apache 2.4 — either works; Nginx is lighter)
  5. Wait for installation to complete, then open Web Station from the main menu
Step 3

Create a virtual host

A virtual host tells Web Station which folder to serve when someone visits your domain or local address.

  1. Open Web Station
  2. Click Web Service Portal in the left sidebar
  3. Click Create → Create Service Portal
  4. Choose Name-based (if hosting on a domain) or Port-based (for local/IP access)
  5. Fill in the details:
    Portal type: Virtual Host
    Hostname: balanceacupuncture.uk (or your domain / local IP)
    Port: 80 (HTTP) — add 443 separately for HTTPS
    Document root: Select the folder where you upload your site files (see Step 4)
    HTTP back-end server: Nginx 1.x
  6. Click Create
For local network access only (not on the public internet), use Port-based hosting. Set port to 8080 or any unused port. Access the site via http://your-nas-ip:8080.
Step 4

Upload the website files

  1. Open File Station on your Synology
  2. Navigate to web (the default Web Station folder — usually at /volume1/web/)
  3. Create a new subfolder, e.g. balance-acupuncture
  4. Upload all the website files into this folder — you can drag and drop in File Station, or use an FTP/SFTP client such as FileZilla
  5. Make sure index.html is at the root of the folder (not inside another subfolder)
  6. Go back to Web Station → your virtual host → set the Document root to this balance-acupuncture folder
Using FileZilla or SFTP? Enable SSH on your Synology (Control Panel → Terminal & SNMP → Enable SSH), then connect with FileZilla using SFTP, your NAS IP, port 22, and your admin credentials.
Step 5

Configure the custom 404 page

To make Nginx serve your custom 404.html instead of its default error page, you need a small config snippet.

  1. In Web Station, select your virtual host and click Edit
  2. Click the Advanced Settings tab
  3. In the Custom configuration box, add:
error_page 404 /404.html;
location = /404.html {
    internal;
}
Apache users: Instead, create a file called .htaccess in your web root folder with: 
ErrorDocument 404 /404.html
Step 6

Enable HTTPS with Let's Encrypt

If your Synology is accessible on the public internet with a real domain name, you can get a free SSL certificate automatically.

  1. Make sure your domain's DNS A record points to your public IP address, and port 80 is forwarded on your router to the Synology
  2. In DSM, go to Control Panel → Security → Certificate
  3. Click Add → Add a new certificate → Get a certificate from Let's Encrypt
  4. Enter your domain name (balanceacupuncture.uk) and email address
  5. DSM will automatically obtain and renew the certificate
  6. Go to Configure and assign the certificate to your Web Station service
  7. In Web Station, add a second listener for port 443 (HTTPS) on your virtual host
Port forwarding required. For Let's Encrypt to work, port 80 must be forwarded from your router to your Synology. Check your router's admin panel (usually at 192.168.1.1) and add a port forwarding rule: external port 80 → Synology IP, port 80. Repeat for port 443.

To force all HTTP traffic to HTTPS automatically, add this to your Nginx custom configuration:

if ($scheme = http) {
    return 301 https://$host$request_uri;
}
Step 7

Set up the contact form (Formspree)

The contact form uses Formspree to send submissions to your email — no server-side code needed.

  1. Go to formspree.io and create a free account
  2. Click + New Form — name it "Balance Acupuncture Contact"
  3. Set the notification email to info@balanceacupuncture.uk
  4. Copy the form endpoint — it looks like: https://formspree.io/f/abcdefgh
  5. Open contact.html in a text editor
  6. Find the line: action="https://formspree.io/f/YOUR_FORM_ID"
  7. Replace YOUR_FORM_ID with your actual form ID, e.g. action="https://formspree.io/f/abcdefgh"
  8. Save the file and re-upload it to your Synology
  9. Test the form — you should receive a confirmation email from Formspree and then the message at your inbox
Free tier limits: Formspree's free plan allows 50 submissions per month. For a small clinic this is typically sufficient. Paid plans start at around $10/month for unlimited submissions.
Alternative — mailto fallback: If you don't want to use Formspree, change the form action to action="mailto:info@balanceacupuncture.uk" and add enctype="text/plain". This opens the user's default email client. It works, but is less reliable and provides no server-side validation.

Ongoing maintenance

Updating the site

To update any page, simply edit the relevant .html file on your computer, then re-upload it to the same location on your Synology via File Station or SFTP. The change goes live immediately — there is no build step or deployment process.

Editing tip. Open .html files in a proper code editor such as VS Code (free). It gives you syntax highlighting, auto-complete and easy search-and-replace — much easier than Notepad.

The shared style.css file controls colours, fonts and layout across the whole site. Changing a colour variable in style.css will update it everywhere instantly.
